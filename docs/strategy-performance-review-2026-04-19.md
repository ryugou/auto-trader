## 2. 問題分析

### 安全 (bb_mean_revert_v1)

- **勝率 63%**: ミーンリバージョン型として正常範囲 (目標 55-70%)
- **R:R 0.79**: 逆転している (勝ち額 < 負け額)。ミーンリバージョンの R:R は 0.5-1.5 が許容範囲だが、1.0 未満は勝率依存が強く脆い
- **問題の本質**: SL 2% が BB 中線までの利幅に対して広すぎる可能性。負けトレードで大きく持っていかれ、勝ちトレードでは中線到達前に反転するケースがある
- **改善方向**: R:R を 1.0 以上に引き上げる (SL を ATR ベースに絞る、or 部分利確で勝ち額を確保)

### 通常 / vegapunk連動 (donchian_trend_v1 / evolve)

- **勝率 27%**: Turtle System の実績ベース (30-45%) を下回る。M5 での偽ブレイクアウト多発が原因
- **R:R 1.51**: Turtle 実績 (5:1-10:1) の 1/3 以下。勝ちトレードが伸びきる前にトレーリング exit (10 バー Donchian) で利益を返している
- **問題の本質**: 日足設計のロジックを M5 に持ち込んでいる。時間足が短すぎてノイズを拾い、かつトレーリング exit が利益を返しすぎる
- **改善方向**: (A) 時間足を 1H-4H に引き上げる、(B) M5 なら追加フィルター (RSI/MACD/出来高等) で偽ブレイクアウトを除外、(C) R:R を 3:1+ に引き上げる exit 設計

### 攻め (squeeze_momentum_v1)

- **勝率 25%**: TTM Squeeze の推奨勝率 (日足で 30-40%) を下回る。M5 + 3 バーの短いスクイーズ期間が原因
- **R:R 1.40**: 設計上の期待 (3:1+) の半分以下。EMA(21) トレーリングで利益を返しすぎ
- **問題の本質**: 通常と同構造。Squeeze バー数 3 は Carter の最低ライン、かつ M5 は推奨外
- **改善方向**: 通常と同じ + スクイーズ検知期間を 6+ バーに延長

## 4. KPI 設定

### 損益分岐の公式

```
期待値 = (勝率 × 平均勝ち) - (負け率 × 平均負け)
期待値 > 0 なら長期的にプラス
```

### 戦略タイプ別 KPI

| KPI | ミーンリバージョン型 (安全) | トレンドフォロー型 (通常/攻め) |
|-----|--------------------------|---------------------------|
| 勝率 | 55% 以上 | 30% 以上 |
| R:R (平均勝ち/平均負け) | 1.0 以上 | 3.0 以上 |
| 期待値 (1トレードあたり) | 0 超 | 0 超 |

### 現状 vs KPI

| 口座 | 勝率 (目標) | 勝率 (実績) | R:R (目標) | R:R (実績) | 期待値 | 判定 |
|------|-----------|-----------|----------|----------|-------|------|
| 安全 | 55%+ | 63% OK | 1.0+ | 0.79 NG | +7.1/trade | PASS (ギリギリ) |
| 通常 | 30%+ | 27% NG | 3.0+ | 1.51 NG | -70.6/trade | FAIL |
| 攻め | 30%+ | 25% NG | 3.0+ | 1.40 NG | -37.1/trade | FAIL |

期待値の計算:
- 安全: (0.63 × 217) - (0.37 × 276) = 136.7 - 102.1 = +34.6 → ÷ n ≈ +7.1/trade (注: サンプル少で統計的信頼性低)
- 通常: (0.27 × 338) - (0.73 × 224) = 91.3 - 163.5 = -72.2/trade
- 攻め: (0.25 × 130) - (0.75 × 93) = 32.5 - 69.8 = -37.3/trade

## 5. 改善優先度

1. **ポジションサイジング**: 口座 95-100% → 1-2% リスクに変更 (全戦略共通、構造的リスク排除)
2. **通常/攻めの R:R 改善**: R:R 3.0+ 達成のための exit 設計見直し (部分利確、ATR ベース SL、時間足変更)
3. **安全の R:R 改善**: R:R 1.0+ 達成のための SL/exit 見直し
4. **通常/攻めの勝率改善**: 偽ブレイクアウト除外フィルター or 時間足引き上げ
