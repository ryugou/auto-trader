# トレード履歴を口座ごとのテーブルに分割する

- 作成日: 2026-04-08
- 対象画面: ダッシュボード `トレード` タブ (`dashboard-ui/src/pages/Trades.tsx`)
- 関連コンポーネント: `dashboard-ui/src/components/TradeTable.tsx`, `dashboard-ui/src/components/PageFilters.tsx`

## 目的

現在のトレード履歴画面は、全口座を横断した1つのテーブルに `口座` カラムを持たせて混在表示している。運用が複数口座（FX ペーパー・暗号資産ペーパー・実口座）に広がり、「口座ごとにどう動いているか」を一目で把握したい場面が増えた。特にペーパートレード口座では「正しくトレードされているか」を空テーブルから検知する必要があり、口座ごとに独立したビューが監視手段として有効になる。

## スコープ

- Trades ページを口座ごとの複数テーブル構成に作り替える
- Trades ページのフィルタを期間のみに絞る（投資種別・口座フィルタは削除）
- `TradeTable` コンポーネントを「特定1口座のトレード一覧を描画する」責務に変更する
- 他ページ（Positions 等）で使われている `PageFilters` は変更しない

## 非スコープ

- 口座ごとのページング（将来の拡張余地として設計上は考慮するが、今回は実装しない）
- 折りたたみによる縦スペース節約
- `TradeTable` のタイムライン展開ロジックの変更
- バックエンド API の変更

## ユーザー体験

### 画面構成

```
┌─ トレード履歴 ─────────────────────────────
│ [期間: 全期間 / 今日 / 1週間 / 1ヶ月]
│
│ ── 暗号資産 ──
│   bitFlyer Paper 100k  ［ペーパー］   評価額 ¥102,345 ／ 残高 ¥100,000 ／ レバレッジ 5x
│   ┌──────────────────────────────────
│   │ <テーブル: 最新10件>
│   └──────────────────────────────────
│
│   bitFlyer Live  ［通常］   評価額 ¥98,000 ／ 残高 ¥95,000 ／ レバレッジ 5x
│   ┌──────────────────────────────────
│   │ <テーブル: 最新10件>
│   └──────────────────────────────────
│
│ ── FX ──
│   OANDA Demo  ［ペーパー］   評価額 $10,120 ／ 残高 $10,000 ／ レバレッジ 25x
│   ┌──────────────────────────────────
│   │ トレードデータがありません
│   └──────────────────────────────────
└────────────────────────────────────────
```

### グルーピングと並び順

- 第1階層: 投資種別（`exchange`）によるグルーピング
  - **暗号資産グループを先**（`bitflyer_cfd` 等）
  - その後 **FX グループ**（`oanda` 等）
- 第2階層: 同一グループ内は口座名の昇順

### テーブル見出し

各テーブル上部に1行の見出しを置く:

- 左: 口座名 + 種別バッジ `［ペーパー］` / `［通常］`
  - ペーパー: 灰色系
  - 通常: 青系（従来 Positions 等で使っている色に揃える）
- 右: メタ情報（順序固定）
  - `評価額 {currency_symbol}{evaluated_balance}`
  - `残高 {currency_symbol}{current_balance}`
  - `レバレッジ {leverage}x`
- 通貨記号は口座の `currency` から決定（JPY → `¥`、USD → `$`、それ以外は記号省略し通貨コードを後置）
- `evaluated_balance` が未取得の場合は `残高` のみ表示（`current_balance` は必須扱い）

### テーブル本体

- 常に最新10件を表示（`per_page=10`, `page=1`）
- ページング UI なし
- カラム: `日時` `戦略` `ペア` `方向` `エントリー` `エグジット` `数量` `PnL` `手数料` `Net PnL` `保有時間` + 展開 chevron
- 現状から削除: `口座`, `種別`
- 行クリックによるタイムライン展開は現状維持
- 該当期間にトレードが0件の口座でも、空テーブルとして描画し `トレードデータがありません` を中央に表示（= 「監視対象口座がそこに存在すること」を保証）

### 期間フィルタ

- 選択肢: `全期間 / 今日 / 1週間 / 1ヶ月`（現状の `PageFilters` と同じ）
- 期間変更時、すべての口座テーブルが再フェッチされる

## 実装方針

### コンポーネント責務

**`dashboard-ui/src/pages/Trades.tsx`**

- 期間 state (`from`, `to`) を保持
- `useQuery(['accounts'], api.accounts.list)` で口座一覧を取得
- 取得した口座を `exchange` でグルーピング（暗号資産 → FX 順）し、各グループ内を口座名で昇順ソート
- 各口座について `<TradeTable accountId={...} from={...} to={...} />` を描画
- 期間フィルタ UI は簡易に同ファイル内インライン化、または `TradesPeriodFilter` として新規小コンポーネントに切り出す（判断は実装時）

**`dashboard-ui/src/components/TradeTable.tsx`**

- props を次の型に変更:
  ```ts
  interface TradeTableProps {
    account: PaperAccount   // 見出し描画に必要な情報を親から直接受け取る
    from?: string
    to?: string
  }
  ```
- 親 Trades.tsx が `useQuery(['accounts'])` で取得した口座を props で渡す（`TradeTable` 内で再取得しない）
- `api.trades.list({ paper_account_id: account.id, from, to, page: '1', per_page: '10' })` を1回呼ぶ
- ページング state およびページャー UI は削除
- `columns` から `paper_account_id`（口座名表示） と `account_type`（種別）を削除
- 見出し (`<header>`) を新設し、口座名 / 種別バッジ / 評価額 / 残高 / レバレッジを描画
- 15秒オートリフレッシュ挙動は現状のダッシュボード既定（`refetchInterval` 等）に従う

**`dashboard-ui/src/components/PageFilters.tsx`**

- 変更なし（他ページで引き続き使用）

### データフロー

```
Trades.tsx
  ├─ useQuery(['accounts'])        ───→ accounts: PaperAccount[]
  ├─ useState: period              ───→ from / to
  │
  └─ groupByExchange(accounts)
        └─ 各 account ごとに
              <TradeTable account={...} from={...} to={...} />
                 └─ useQuery(['trades', {accountId, from, to}])
                        ───→ trades 最新10件
```

- N 口座に対して N 個の並列 `trades` クエリが発生する。口座数は現状一桁で十分許容可能。
- React Query のキー設計により、期間変更時のみ全テーブルが再フェッチされる。

### ソート仕様

- `exchange` 値の分類:
  - 暗号資産グループ: `bitflyer_cfd`, その他暗号資産系（必要に応じ将来追加）
  - FX グループ: `oanda`, その他 FX 系
- 分類ロジックは Trades.tsx 内のローカル `exchangeGroup(exchange: string): 'crypto' | 'fx' | 'other'` で実装
- 未分類 (`other`) が出た場合は FX グループの後ろに表示（運用中に気づけるように）

## エッジケース

- **口座が1件も無い**: `口座が登録されていません` と中央に表示
- **特定口座の `evaluated_balance` が未取得**: 評価額を見出しから省略し、残高とレバレッジのみ表示
- **空テーブル**: 各口座ごとに現状の `トレードデータがありません` をそのまま出す
- **期間フィルタ変更直後**: 各テーブルで個別にローディング表示
- **口座の `currency` が JPY/USD 以外**: 記号を付けず通貨コードを後置（例: `12,000 EUR`）

## テスト観点

- 口座グルーピング・ソート順が暗号資産 → FX、各グループ内は口座名昇順になること
- 種別バッジがペーパー/通常で色分けされること
- 空口座でも見出しとテーブル骨格が描画されること
- 期間フィルタ変更で全テーブルが再フェッチされること
- 展開タイムラインが口座ごとに独立して動作すること
- `口座` / `種別` カラムが行から消えていること

## 既存コードへの影響

- `dashboard-ui/src/pages/Trades.tsx`: 大幅に書き換え
- `dashboard-ui/src/components/TradeTable.tsx`: props 型変更、ページング削除、カラム削除、見出し追加
- `dashboard-ui/src/components/PageFilters.tsx`: 変更なし
- 他ページ（Positions 等）: 影響なし

## 将来の拡張余地

- 口座ごとのページング: `TradeTable` に `page` state を追加するだけで実現可能な構造にしておく
- グループ折りたたみ: 口座数が増えた場合に検討
- ソート・フィルタの拡張（ペア絞り込み等）: 必要に応じて各テーブルヘッダーに追加
